# Project Onboarding Template

## Purpose

Use this document to explain how Codex should understand and operate within a specific repository after the starter kit has been adopted.

## Repository Summary

Describe what the repository is for, who maintains it, and what kind of work Codex is expected to perform here.

## Working Constraints

List important boundaries such as safety expectations, review practices, release constraints, or non-goals.

## Validation Paths

List the validation commands or checks that matter most in this repository.

## Architecture Notes

Summarize the important parts of the repository structure that Codex should learn first.

## Local Skill Additions

Document any project-local skills or pack-based extensions that exist in addition to the canonical base skills.

## Handoff Expectations

Describe what a complete Codex task handoff should include for this repository.
