# Skills Manifest Template

Use this template to register available skills in an adopting repository.

## Status Values

- `active`: approved and recommended for normal use
- `experimental`: available for trial use but not yet stable
- `deprecated`: still present but scheduled for replacement or removal
- `retired`: kept only for historical reference and not recommended for use

## Skills

| Skill | Layer | Status | Path | Purpose |
| --- | --- | --- | --- | --- |
| example-skill-name | atomic | active | `.codex/skills/atomic/example-skill-name/SKILL.md` | Replace this row with the skill's concise purpose. |

## Maintenance Notes

- Keep this file aligned with the real skill folders.
- Update status when a skill changes maturity.
- Reflect additions, renames, deprecations, and retirements in the same change.
