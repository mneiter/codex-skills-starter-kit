# Codex Agent Rules

## Purpose

These rules illustrate how an adopting repository can localize Codex guidance without changing the canonical base skills.

## Working Style

- Learn the repository shape before editing.
- Prefer small reviewable changes.
- Keep project guidance in `.codex/skills/project/` instead of modifying the shared base skills.

## Documentation Expectations

- Keep the local onboarding document current.
- Update the local manifest and index when local skills change.

## Validation Expectations

- Run the relevant repository checks when possible.
- Report blockers or skipped checks explicitly.
