# Minimal Project Onboarding

## Repository Summary

This example repository illustrates how a team can adopt the starter kit and then add repository-specific guidance beside the canonical base layer.

## Local Constraints

- keep the canonical base skills unchanged unless the starter kit itself is being updated
- store local knowledge in project-local skills
- keep manifests and indexes current when local skills change

## Validation Paths

Document the repository's real validation commands here after adoption.

## Architecture Notes

The base layer comes from the starter kit. Local additions live beside it and should not change the meaning of canonical base skills.

## Local Skill Additions

- `project-context`

## Handoff Expectations

Local handoffs should distinguish between generic starter-kit guidance and project-specific rules.
