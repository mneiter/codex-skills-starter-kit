# Skill Naming Conventions

## Naming Rules

- Use lowercase letters, digits, and hyphens only.
- Keep names short, stable, and reusable.
- Prefer verb-led or action-oriented phrases when they improve clarity.
- Name the skill folder exactly after the skill name.

## Layer Alignment

Choose names that reflect the skill's role in the canonical layer model:

- orchestration names should imply coordination, intake, routing, or handoff
- guardrail names should imply protection, review, verification, or boundaries
- atomic names should imply one focused task

## Stability Guidance

- Avoid names tied to one project, team, framework, or vendor.
- Avoid temporary phrasing that will age quickly.
- Prefer names that can survive reuse across many repositories.

## Good Naming Outcomes

A good skill name should help another engineer understand:

- what the skill does
- roughly when it should trigger
- how it differs from neighboring skills
